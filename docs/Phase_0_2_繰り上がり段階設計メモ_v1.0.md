## 💡 人間開発者の提案（原文）

> 個人的な感想ですが足し算基本（10までの足し算、足して10になる足し算、
> 穴埋め算 3＋？＝10 のような）
> 繰り上がりの前に簡単な繰り上がりの説明（子供でもわかるように丁寧な）があって
> 練習（最初が9固定の繰り上がり足し算、最初が8固定の足し算、最初が7か6の足し算）
> 応用問題で初めて繰り上がりのランダム問題。
> くらい親切でも子供にはいいような。

## 📚 提案の整理

### たしざん（2）繰り上がり の段階設計

```
ステップ1: 基礎固め（Phase 0.1 で実施済み）
  ├─ M1-05 きほん: 10までの足し算
  │    10+n 系の足し算
  │    足して10になる足し算（補数の感覚）
  │    穴埋め算（□ + 5 = 10 など）
  └─ M1-05 おうよう: 文章題・式の選択

ステップ2: 繰り上がりの導入説明（Phase 0.2 冒頭）
  ├─ 「10をつくる」という考え方の丁寧な説明
  │    例: 9 + 3 → 「9にいくつたせば10？」→「1」→「10 + 2 = 12」
  ├─ 子どもでもわかる言葉・絵で説明
  └─ 実際にやってみよう（体験学習的なウォームアップ問題）

ステップ3: 9固定の繰り上がり（最も簡単）
  ├─ 9 + 2, 9 + 3, 9 + 4 ... 9 + 9
  ├─ 9は「あと1で10」なので補数が最もわかりやすい
  └─ 問題数: 8〜10問（9+2〜9+9）

ステップ4: 8固定の繰り上がり
  ├─ 8 + 3, 8 + 4 ... 8 + 9
  ├─ 8は「あと2で10」
  └─ 問題数: 7〜8問

ステップ5: 7・6固定の繰り上がり
  ├─ 7 + 4 〜 7 + 9 / 6 + 5 〜 6 + 9
  ├─ 難易度が上がる（補数が3や4になる）
  └─ 問題数: 8〜10問（混合）

ステップ6: ランダム繰り上がり（応用）
  ├─ 全ての繰り上がりパターンをランダムに出題
  ├─ 文章題も含む
  └─ 問題数: 15問（標準）
```

## 📝 問題設計の方針（Phase 0.2 向け）

### 繰り上がりの導入説明（ウォームアップ問題の例）

```javascript
// 繰り上がりの「考え方」を誘導する問題例

{
  question: '9 + 3 を かんがえよう。\n9に いくつ たすと 10に なる？',
  choices: ['1', '2', '3', '4'],
  correctAnswer: '1'
  // → 「では 3は 1と □に わかれる」へ続く
},

{
  question: '9 + 3 = ?\n（ヒント: 9 + 1 = 10、10 + 2 = ?）',
  choices: ['10', '11', '12', '13'],
  correctAnswer: '12'
}
```

### 9固定ワールドの問題例

```javascript
{ question: '9 + 2 = ?', choices: ['10', '11', '12', '13'], correctAnswer: '11' },

{ question: '9 + 8 = ?', choices: ['15', '16', '17', '18'], correctAnswer: '17' }
```
