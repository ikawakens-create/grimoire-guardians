# 計画書：ひっ算モード実装

---

## 🎯 目的

タブレット・PC で「ひっ算（筆算）」を完結させる。
難易度に合わせてワールドを分割し、B+C（桁別入力）→ A（全桁一括）を
別ワールドとして段階的に学べる構成にする。

---

## 📐 2つのモード

### モードB+C（桁別入力）
```
┌─────────────┐
│  ¹           │  ← 繰り上がり（正解後にふわっとアニメーション）
│    ２ ３    │
│  ＋    ８   │
│  ───────    │
│    □ □    │  ← 一の位から順に4択で答える
└─────────────┘
  [1]  [2]  [3]  [4]
```
1. 一の位を4択で答える
2. 正解 → 繰り上がりがあれば「１」がふわっとアニメーション表示（0.8秒）
3. 十の位を4択で答える
4. 全桁正解 → 正解フィードバック → 次問
5. どちらかで間違え → 即不正解フィードバック → 次問

### モードA（全桁一括）
```
┌─────────────┐
│    ２ ３    │
│  ＋    ８   │
│  ───────    │
│    □ □    │
└─────────────┘
 [29]  [30]  [31]  [32]  ← 答え全体を4択
```
縦式を表示しつつ、答え全体を4択で一発選ぶ。

## 📝 問題データ形式

```js
// B+C モード（桁別入力）
{
  id: 'M1-10a-1-Q01',
  unitId: 'M1-10a-1',
  step: 1,
  type: 'hitsuzan',
  operator: '+',
  operand1: 9,
  operand2: 3,
  correctAnswer: '12',
  hitsuzanMode: 'digit-by-digit'
}

// A モード（全桁一括）
{
  id: 'M1-10a-2-Q01',
  unitId: 'M1-10a-2',
  step: 1,
  type: 'hitsuzan',
  operator: '+',
  operand1: 9,
  operand2: 3,
  correctAnswer: '12',
  hitsuzanMode: 'full-answer'
}
```

HitsuzanRenderer が自動導出するもの：
- `onesDigit` — 一の位の正解
- `tensDigit` — 十の位の正解
- `hasCarry` / `hasBorrow` — 繰り上がり・繰り下がりの有無
- 選択肢4択 — 正解 ± 1〜3 の誤答を自動生成

## 🖥️ 画面フロー（digit-by-digit）

```
問題表示（縦式, 両桁□）
  ↓
一の位の4択表示
  ↓ 正解
  繰り上がり/繰り下がりアニメーション（あれば 0.8秒）
  ↓
十の位の4択表示
  ↓ 正解
正解フィードバック → 次問 or 結果画面

  ↓ 不正解（一の位 or 十の位どちらかでも）
不正解フィードバック → 次問 or 結果画面
```

---

## ⚠️ 気をつけること

1. **GameStore の記録タイミング**：両桁確定後に1回 `recordAnswer()` を呼ぶ（途中状態は記録しない）
2. **イベントトリガー**：1問扱いのため既存の EventManager に影響なし
3. **既存ユニット（M1-10a 等）はそのまま残す**：旧ワールドを削除せず、新ユニット（M1-10a-1/2）を追加する形で展開する
4. **validateQuestion の後方互換**：`type:'hitsuzan'` は `choices` フィールド不要（distractorPool モードと同様の扱い）
5. **プロトタイプは M1-14c-1 のみ**：まず1ファイル+1ワールドで動作確認してから全展開
