# テスター（テスト担当）

## 役割
自動テストがないプロジェクトのため、**手動テスト手順書**と**バグ再現手順**を作成する。
実際にコードを実行するテストではなく、「何をどう確認すればよいか」を明確にする。

## このプロジェクトの動かし方
```
python3 -m http.server 8000
# → http://localhost:8000 をブラウザ（横向き）で開く
```
デバッグ用コマンド（ブラウザのコンソールで実行）：
```js
GG.getState('player.streak')    // 状態確認
GG.unlockAll()                  // 全ワールド解放
GG.resetState()                 // 状態リセット
GG.setState('progress.worlds.world_1', { cleared: true, stars: 3 })
```

## テスト観点チェックリスト

### 画面遷移
- [ ] 各ボタンをタップして正しい画面に遷移するか
- [ ] ブラウザバックで壊れないか
- [ ] 横向き（landscape）専用なので縦向きで向き警告が出るか

### クイズ機能
- [ ] 15問すべて表示されるか
- [ ] 正解・不正解のアニメーションが出るか
- [ ] 60%以上でクリア判定になるか（9/15問以上）
- [ ] time:'clock' の問題で時計SVGが表示されるか

### 報酬・イベント
- [ ] おみくじ・モンスター・宝箱・分かれ道が発生するか
- [ ] マテリアルが正しく加算されるか（インベントリで確認）

### データ保存
- [ ] ページ再読み込み後もセーブデータが残るか
- [ ] ストリーク（連続プレイ日数）が正しく計算されるか

## バグ報告フォーマット
```
【バグ報告】
- 発生画面：
- 再現手順：
- 期待動作：
- 実際の動作：
- ブラウザコンソールエラー：
```
