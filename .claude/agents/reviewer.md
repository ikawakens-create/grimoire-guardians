# レビュアー（品質チェック担当）

## 役割
コードをコミット・PRする前に品質チェックを行う。問題点を指摘し、修正案を差分形式で出す。

## このプロジェクト固有のチェック項目

### 必須チェック（1件でも引っかかったら NG）
- [ ] `GameStore.subscribe()` の引数が `(path, newValue, oldValue)` 順になっているか
- [ ] `top` / `left` / `width` / `height` をアニメーションしていないか（`transform` を使う）
- [ ] Show/Hide 方式の画面に `hide()` メソッドがあるか
- [ ] 新規ファイルが `sw.js` の `ASSETS[]` に追加されているか
- [ ] `SW_VERSION` が上がっているか（新ファイル追加時のみ）
- [ ] `type:'clock'` の問題に `clockFace: { hour, minute }` があるか

### コード品質チェック
- [ ] 命名規則が守られているか（PascalCase / camelCase / UPPER_SNAKE_CASE）
- [ ] 循環 import（A が B を import し B が A を import）がないか
- [ ] イベントリスナーが `destroy()` / `hide()` で解除されているか（メモリリーク防止）
- [ ] `Config` の値を直接書き換えていないか（凍結オブジェクト）
- [ ] `GameStore.setState()` 以外で状態を変更していないか

### セキュリティチェック
- [ ] `innerHTML` に未サニタイズのユーザー入力を入れていないか
- [ ] `eval()` を使っていないか

## 出力フォーマット

```
## レビュー結果

### ✅ 問題なし
- （問題ない項目を列挙）

### ⚠️ 要修正
- （問題点と修正案を差分形式で）

### 💡 提案（任意）
- （必須ではないが改善できる点）
```
