# 設計判断ログ

重要な設計上の決定をここに記録する。「なぜそうしたのか」を残すことで、後から迷わないようにする。

---

## 2026-03-05: TownScreen を SVG ホットスポット方式に変更
**決定内容**: カードボタン方式 → SVG マップ上のホットスポットタップ方式
**理由**: よりゲームらしい世界観の演出のため
**影響**: TownScreen.js の全面刷新、画像アセット追加

## 2026-03-03: GameStore.subscribe の引数順修正
**決定内容**: `(newValue, path)` → `(path, newValue, oldValue)` に修正
**理由**: ルーターのサブスクライバーが path を第1引数として期待していたため
**影響**: index.js のルーターが正常動作するように修正済み

## 2026-02-26: 家ビルドを6セクション構成に確定
**決定内容**: floor1 / garden / floor2 / exterior / floor3 / tower の6セクション
**理由**: ワールドクリア数に応じて段階的に解放するゲームループを作るため
**影響**: HouseManager.js・GameStore の house.sections に実装済み

## 2026-02-23: 画面ライフサイクルを2方式に分離
**決定内容**: Create/Destroy（クイズ系）と Show/Hide（施設系）の2パターン
**理由**: 施設画面は状態を保持したまま切り替えたいため
**影響**: index.js のルーター設計・全施設画面の実装方針
