## 核心的な問題（改訂で追加）

### 「暗記前の子供がいきなりつまずく」問題

旧プランのStep 1 は `2 × 3 = ？` という**純粋な暗記テスト**だった。

これはすでに九九を覚えた子供向け。
**初めて2のだんを習う子供**は答えを導く手がかりがなく、
最初から間違え続けて**ゲームが嫌になる**。

---

## 改訂後のステップ設計

### 3ステップの役割

| ステップ | 名前 | 目的 | 仕掛け |
|---------|------|------|--------|
| **Step 1** | じゅんばんに おぼえよう | 初めての子が自分で考えて答えられる | **前の式を問題文にヒントとして載せる** |
| **Step 2** | おぼえたかな？ | 記憶から引き出す練習 | ヒントなし・順不同・隣のだんの選択肢 |
| **Step 3** | おうよう | 本当に使いこなせるか | 逆向き（□×N=X）または場面文章題 |

---

### Step 1 の設計詳細：「ヒントつき連続出題」

**前の式の答えをヒントとして問題文に書く**。
暗記ゼロでも「2のだんは2ずつ増える」というルールで答えが出せる。

```
例（2のだん）：
Q01: 「2×1=2です。では、2×2=？」   → [2, 4, 6, 8]
Q02: 「2×2=4です。では、2×3=？」   → [4, 5, 6, 8]
Q03: 「2×4=8です。では、2×5=？」   → [8, 9, 10, 12]
Q04: 「2×6=12です。では、2×7=？」  → [12, 13, 14, 16]
```

子供の思考：「2×4=8、8に2を足すと10！だから2×5=10」
→ 暗記不要で自分の力で正解できる → 成功体験 → 覚える気が出る

ポイント：
- ヒントは「その一つ前の式と答え」を使う
- 選択肢は正解の前後の数字（例：14なら12・13・15・16）
- Step 1 の9問はすべてこの「ヒントつき」形式

---

### Step 2 の設計詳細：「ヒントなし・バラバラ出題」

Step 1 と**まったく同じ乗算**を、今度はヒントなしで出す。

```
例（2のだん）：
Q10: 「2×7=？」  → [12, 14, 16, 18]  （隣のだんの答えを選択肢に）
Q11: 「2×4=？」  → [6, 7, 8, 10]
Q12: 「2×9=？」  → [16, 17, 18, 20]
```

選択肢の作り方（重要）：
```
❌ 悪い例：[13, 14, 15, 16]（連番で直感で正解できる）
✅ 良い例：[12, 14, 16, 18]（2のだんの近くの答えを使う）
✅ より良い例：[12, 14, 21, 16]（3×7=21 など別のだんを混ぜる）
```

---

### Step 3 の設計詳細：「おうよう（逆向き or 文章）」

2種類を混在させる：

**逆向き型**：
```
「2×□=14、□にあてはまる数は？」 → [5, 6, 7, 8]
```

**場面文章型**：
```
「クッキーが 2まいずつ 5ふくろ あります。ぜんぶで なんまいですか？」
→ [7まい, 8まい, 10まい, 12まい]
```

Step 3 は 3問だけ。「難しい」で終わらず、後味よく終わることを優先。

## プール構成と pick 数

```
問題数: 9問(Step1) + 9問(Step2) + 3問(Step3) = 21問
pick:   4問(Step1) + 3問(Step2) + 2問(Step3) = 9問

stepConfig: [
  { step: 1, pick: 4 },  ← ヒントつき（やさしい）
  { step: 2, pick: 3 },  ← ヒントなし（ふつう）
  { step: 3, pick: 2 }   ← おうよう（むずかしい）
]
```

プール > pick にすることで、**毎回異なる9問**が選ばれる。
2回目・3回目でも新鮮に感じられる。

---

## 「2回目に挑戦したとき」のシナリオ

```
1回目：ヒントを頼りながら正解 → 「できた！」という体験
2回目：前回と違う9問が出る → Step 1 のヒント問題でまた自信をつける
3回目：だんだん Step 2 のヒントなしも正解できるようになってくる
4回目以降：Step 3 の逆向き・文章題にも挑戦できるようになる
```
