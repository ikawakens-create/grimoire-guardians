## なぜ今の九九ファイルだと子供がつまずくか

### 問題1：毎回まったく同じ問題が出る

現在の設定は `pool=9問、pick=9問` = **毎回全問・常に同じ**。

子供は「2×1=2、2×2=4…」と**順番で覚える**だけになり、
バラバラに出題されると答えられない（本当の理解になっていない）。

### 問題2：「答えられる形式」しかない

`2 × 3 = ？` だけ。
でも実際のテストや日常では：
- `2 × □ = 6`（□に入る数は？）
- `くりが 2こずつ 3ふくろ → ぜんぶで？`（場面理解）

この2つができないと、次のUnit（文章題・総まとめ）で突然つまずく。

### 問題3：選択肢が「なんとなく正解できる」

現在の例：`2 × 5 = ？` → 選択肢 `[8, 9, 10, 12]`
→ 「一番中間そうな数」で正解できてしまう。

**隣のだんの答え**を混ぜると識別力が上がる：
- `2×5=10` vs `3×5=15` vs `2×6=12` → `[10, 12, 15, 20]`

## 設計方針（子供がつまずかないために）

### 原則1：プール > 出題数にする（毎回違う問題を体験させる）

```
pool: 21問、pick: 9問  ← 毎回ランダムに9問が選ばれる
```

### 原則2：ステップで難易度を分ける

| ステップ | 形式 | 難易度 | pick数 |
|---------|------|--------|--------|
| Step 1 | `N×□=？`（順方向） | やさしい | 4問 |
| Step 2 | `N×□=X、□は？`（逆向き） | ふつう | 3問 |
| Step 3 | 場面・文章から式を立てる | むずかしい | 2問 |

→ 合計9問。やさしい問題から始まり、自信をつけながら難しくなる。

### 原則3：選択肢は「隣のだんの答え」を混ぜる

例：`3 × 4 = ？`
```
❌ 悪い選択肢: [11, 12, 13, 14]（数字を並べるだけ）
✅ 良い選択肢: [12, 15, 16, 9]（3×4=12, 3×5=15, 4×4=16, 3×3=9）
```

隣のだんや混同しやすい組み合わせを使うことで、本当に覚えているかを測れる。

### 原則4：Step 3 は「場面」のある文章題

`○こずつ △人分` の形。単純計算だけでなく、
「どの九九を使うか」を判断する力をつける。
（M2-10jの文章題へのブリッジになる）

## ファイル別実装内容

### M2-10a〜h（各段）共通フォーマット

```
問題数: 9問(Step1) + 9問(Step2) + 3問(Step3) = 21問
pick:   4問(Step1) + 3問(Step2) + 2問(Step3) = 9問

stepConfig: [
  { step: 1, pick: 4 },
  { step: 2, pick: 3 },
  { step: 3, pick: 2 }
]
```

**Step 1（順方向・9問）**: `N × 1 〜 N × 9 = ？`
→ 選択肢は「隣のだんの答え」を2つ以上含む

**Step 2（逆向き・9問）**: `N × □ = X` または `□ × N = X`
→ 9問それぞれ `□` の位置を変える。答えは1〜9。

**Step 3（場面文章・3問）**:
- 場面問題（「〇こずつ△こ、ぜんぶで？」）
- 図の読み取り（よこ○・たて△のタイル）
- 少し難しい応用（かけた後に足す・引くなど1ステップ多い計算）
