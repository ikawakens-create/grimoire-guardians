# /plan — タスク計画

新機能や修正の計画を立てる。**このフェーズではコードを書かない**。

## 使い方
```
/plan [やりたいこと]
例: /plan 新しいワールド「M2-01」を追加したい
例: /plan 農場画面でハーベストボタンが押せないバグを直したい
```

## アーキテクト（設計担当）として実行する手順

1. 関連ファイルを読んで現状を把握する（コードは書かない）
2. 影響範囲を洗い出す
3. 以下の設計書フォーマットで出力する
4. 「この計画で進めますか？」と確認を取る

## 出力フォーマット

```
## 計画書：[やりたいこと]

### 🎯 目的
（何を実現するか・なぜ必要か）

### 📁 変更するファイル
| ファイル | 変更内容 |
|----------|----------|
|          |          |

### 🗄️ 状態の変更（GameStore）
（新しく追加する state のパス）

### 🖥️ 画面フロー
（どの画面からどう遷移するか）

### ⚠️ 気をつけること
（既存コードへの影響・注意点）

### ⏱️ 作業ステップ（順番）
1.
2.
3.

---
この計画で進めますか？（はい/修正して）
```
