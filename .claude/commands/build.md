# /build — フル開発フロー

計画承認後に実装・レビューまでを順番に行う。

## 使い方
```
/build [計画書またはタスク内容]
例: /build 農場画面のハーベストボタン修正
```

## 実行手順（並列処理なし・順番に実行）

### Step 1: 計画確認
- `/plan` で作った計画書があれば読み込む
- なければ簡単な計画を立てて確認を取る

### Step 2: 実装（デベロッパー担当）
- 差分のみ出力しながら実装する
- 1ファイルずつ変更して確認を取る
- `.claude/agents/developer.md` のルールに従う

### Step 3: 自動チェックリスト（レビュアー担当）
実装後に以下を確認：
- [ ] `GameStore.subscribe()` の引数順（path, newValue, oldValue）
- [ ] アニメーションが transform 使用か
- [ ] 新ファイルを sw.js の ASSETS[] に追加したか
- [ ] SW_VERSION を上げたか

### Step 4: 動作確認指示
```
確認手順：
python3 -m http.server 8000
→ http://localhost:8000 で [確認内容] をテストしてください
```

### Step 5: コミット
問題なければコミットメッセージを提案する。
