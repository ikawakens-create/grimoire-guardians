# /parallel — 並列開発（Agent Team）

⚠️ **このコマンドは API 使用量が多くなるため、原則使用しない。**
明示的に「並列でやって」と指示があった場合のみ実行する。

## 使い方
```
/parallel [並列で行う複数タスク]
例: /parallel 農場画面のバグ修正 と ショップ画面のUIブラッシュアップ
```

## 向いているケース
- 互いに依存しない独立したファイルの変更が複数ある
- 例：別々の画面ファイルを同時に修正する
- 例：複数の問題ファイル（M1-XX.js）を同時に作成する

## 向いていないケース（直列で実行する）
- GameStore や Config など共有ファイルを変更する場合
- 一方の結果が他方の入力になる場合
- 通常の開発（並列は例外）

## 実行手順

並列実行する場合は各タスクを明確に分割し、
それぞれ独立していることを確認してから実行する。

```
タスクA: [ファイル名] → [変更内容]
タスクB: [ファイル名] → [変更内容]
→ 並列実行してもよいか確認：はい / やはり直列で
```
